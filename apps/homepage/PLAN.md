## CEO Review — Sections 1-10

### Section 1: Architecture Review

Static HTML/CSS. No backend. Architecture:

```
[Browser]
  └─▶ index.html (single page, anchor nav)
       ├─▶ assets/production.css (all styles, CSS vars)
       ├─▶ assets/downloads/ (pitch deck PDF, case studies, team bios)
       ├─▶ Google Fonts (external, display=swap)
       └─▶ [Director photos, poster images — gradient placeholders initially]
                 ↓ replace with real images post-production

Investor Library gate (display-only, not security):
  └─▶ JS: if (localStorage.email) → show download links
  └─▶ Email form → localStorage.email = submitted → links visible
       (No backend. No credentials. Gate = UX flow, not access control.)
       (Pitch deck is promotional — anyone can request it by email.)
```

**Language toggle mechanism:**
- Default: `<html lang="ko">` (KO-first)
- Toggle: JS swaps `data-ko` / `data-en` text content via DOM
- State: `sessionStorage.lang = 'ko' | 'en'` (persists per tab)
- No separate HTML files needed

**Image strategy:**
- Initially: CSS gradient placeholders (gradient-a, gradient-b, gradient-c classes)
- Post-production: replace with actual director photos + poster images in /assets/
- Lazy loading via `loading="lazy"` on below-fold images

**Deployment:** Static files on Netlify/Vercel (one-click deploy from git). TBD — add to TODOS.md.

**Scaling:** Static files serve from CDN. Investor Library PDFs in /assets/downloads/.

**Rollback:** `git revert` of single commit. Instant.

### Section 2: Error & Rescue Map

```
CODEPATH                  | WHAT CAN GO WRONG              | EXCEPTION      | RESCUED? | USER SEES
--------------------------|--------------------------------|-----------------|----------|-----------
Google Fonts CDN          | Timeout / 404                   | —              | N/A      | Fallback system fonts
Image assets              | 404 (missing poster photo)     | —              | N/A      | Gradient placeholder (gradient-a/b/c)
Form (contact)            | Empty fields                    | —              | N        | HTML5 validation message
Form (contact)            | mailto: fails (no email client) | —              | N        | Manual email displayed
Investor Library gate     | JS disabled                    | —              | N        | Library hidden (graceful)
Language toggle           | JS disabled                    | —              | N        | Default language shown
```

No backend → minimal error surface. Form uses HTML5 validation + mailto fallback (no server dependency).

### Section 3: Security & Threat Model

- **No user data collected** (static page, contact form is mailto:)
- **No new attack vectors** — pure static content
- **Investor Library gate**: email-based access (display only, no credentials stored)
- **Input validation**: HTML5 `required`, `type="email"` on form fields
- **No secrets** in codebase
- **External resources**: Google Fonts CDN (trusted), image URLs (user-supplied, no user injection)

Threats: None beyond standard static site risks.
