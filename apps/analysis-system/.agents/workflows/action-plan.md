# CROS와 PDCA는 효율적인 프로젝트 관리와 지속적인 품질 개선을 위한 핵심 프레임워크입니다.

## CROS (Cross-functional, Result-oriented, Optimized, Systemic)
 CROS는 주로 조직의 운영 효율성과 협업을 극대화하기 위한 전략적 프레임워크로 활용됩니다. (조직에 따라 구체적인 정의는 다를 수 있으나, 일반적으로 아래의 4가지 가치를 지향합니다.)

> C (Cross-functional) - 다기능 협업: 개발, 기획, 디자인 등 서로 다른 직군이 하나의 목표를 위해 긴밀하게 협력하는 구조입니다. (예: 애자일 스쿼드 구성)

> R (Result-oriented) - 결과 중심: 단순한 활동(Activity)이 아닌, 실질적인 비즈니스 가치와 결과물(Outcome)에 집중합니다.

> O (Optimized) - 최적화: 자원 배분과 워크플로우를 분석하여 낭비를 제거하고 효율을 극대화합니다.

> S (Systemic) - 체계적 접근: 단편적인 해결이 아니라 전체 시스템의 관점에서 문제를 바라보고 구조화된 해결책을 제시합니다.

## PDCA (Plan-Do-Check-Act)
 PDCA는 업무의 품질을 지속적으로 개선하기 위한 반복적인 4단계 관리 사이클입니다. 에드워즈 데밍에 의해 대중화되어 '데밍 사이클'이라고도 불립니다.

> P (Plan) - 계획: 목표를 설정하고 이를 달성하기 위한 프로세스를 설계합니다. (예: 요구사항 분석, 스프린트 계획)

D (Do) - 실행: 계획된 프로세스를 실제로 수행합니다. (예: 코드 작성, 기능 구현)

C (Check) - 점검: 실행 결과를 측정하고 원래 목표와 비교하여 차이점을 분석합니다. (예: 코드 리뷰, QA 테스팅, 성능 모니터링)

A (Act) - 조치: 점검 결과에서 발견된 문제점을 해결하거나, 성공적인 프로세스를 표준화하여 다음 사이클에 반영합니다. (예: 리팩토링, 회고를 통한 프로세스 개선)
